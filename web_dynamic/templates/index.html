Choice Crafter Use

The online Choice Crafter app is a decision making tool that can be used by engineers, researchers and students alike, it is based on the basic multi-criteria decision making(MCDM) concept.

If you are in a situation where you have to decide on one out of many set of optimal solutions or alternative, then this app is for you, for instance an engineer optimizing the wing of an aircraft or researcher working on optimizating machining operation which are probably two objective problem and being solve with maybe multi objective optimization methods like NSGAII, would result in a set of optimal solutions on the pareto front, or better still, you wanting to decide on the best cell phone to purchase from a set of phones, then you are on the right web app, because with your criteria, performance values and type of criteria Choice Crafter is about to process your decison matrix and come up with the base alternative based on the MCDM concept.

The MCDM concept

When faced with set of optimal solution, in MCDM you resolve the solutions into a decision matrix, as show in the the example below

Normalized decision matrix

The decison matrix is then normalized based on the type of criteria, that is: beneficial and non-beneficial Criteria

  • Benficial criteria is one whose higher value is desire
  • Non beneficial criteria is one whose lower value is desire, for instance "cost"
if a criteria is beneficial, it normalized value would be computed as the performance value of a particular alternative divide by the maximum performance value of a particular criteria in it column, else if it a non non-beneficial criteria the minimum performance value of a particular criteria in it column is divided by the each performance value in that criteria column.

  • beneficial = Xij/max(Xij)
  • non-beneficial = min(Xij)/Xij

Weightage normalized decision matrix

After the generation or computation of the normalized decision matrix, the weightage normalized decision matrix is computed, in Choice Crafter app the equal weightage is assummed for all criteria, so if the numbers of criteria is not even, then the logic to handle such situation is handled, therefore in this example the number of criteria is even (4), so our assigned criteria weightage value would be 25% or 0.25 as the sum of weight must always sum up 100%, so for instance applying the weightage to the speed criteria column we have: 0.8889 * 0.25 = 0.2222, 0.9259 * 0.25 = 0.2315, 1.0 * 0.25 = 0.2500, this is done for the rest of the criteria to obtain the weightage normalized decision matrix.

performance Score

the performance score for each alternative is computed using the weightage normalized matrix, where each alternative this the performance value row wise is summed up to obtain the performance score for each alternative, for instance the second alternative performance score is computed as follows: 0.2315 + 0.2500 + 0.2310 + 0.2500 = 0.9636, the performance score for the first and the third alternative are 0.7178 and 0.7361 ,therefore the alternative with the highest performance score is returned as the best alternative or choice, in this example the highest is 0.9636 and it is of the second alternative [1250, 0.5, 1.3, 0.1]
